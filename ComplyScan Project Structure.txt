Architectural Principles
Separation of concerns: Each directory has a single, well-defined responsibility.
Modularity: Components (OCR, parsing, matching, scoring, reporting) are isolated and replaceable.
Scalability: New rule sets, document types, or services can be added without major restructuring.
Maintainability: Documentation, tests, data, and source code are organized into dedicated modules.
Extensibility: Supports adding new compliance domains (beyond HIC and PRE), OCR engines, AI models, report formats, and frontend features with minimal impact on existing code.
